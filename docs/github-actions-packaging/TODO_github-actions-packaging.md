## 配置要求

### 环境变量
以下环境变量可能需要在GitHub仓库设置中配置：
- `GITHUB_TOKEN`: 用于上传artifact和创建Release
- 任何构建过程中需要的敏感信息

### 权限配置
确保GitHub Actions具有以下权限：
- 读取代码仓库
- 上传artifact
- 创建Release(如果需要自动发布)

## 操作指引

### 测试工作流
1. 推送代码到main分支或创建PR到main分支
2. 在GitHub Actions页面查看工作流执行状态
3. 验证构建产物是否正确生成和上传

### 下载构建产物
1. 在GitHub Actions页面找到对应的workflow运行记录
2. 在Artifacts部分下载对应平台的构建产物

### 故障排除
1. 如果构建失败，首先查看构建日志以确定错误原因
2. 检查build.sh脚本在对应平台是否能本地运行
3. 验证GitHub Actions运行器环境是否满足构建要求
