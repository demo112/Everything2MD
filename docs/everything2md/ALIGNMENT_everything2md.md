# ALIGNMENT: Everything2MD 文档转换项目

## 1. 项目背景和需求理解

### 原始需求
用户需要一个能够将 doc、docx、ppt、pptx 格式的文档统一转换为 Markdown 格式的工具。要求以流水线的形式，通过连续转换将所有格式统一转换为 md。

### 边界确认
- 支持的输入格式：.doc, .docx, .ppt, .pptx
- 输出格式：.md (Markdown)
- 需要集成的组件：LibreOffice + Pandoc + pptx2md
- 项目要求：简单、纯净，fork这些组件到项目中独立路径下，不侵入源项目的任何代码

### 需求理解
1. 用户希望有一个统一的工具来处理多种办公文档格式到Markdown的转换
2. 要求采用流水线处理方式，即不同格式可能需要不同的处理路径，但最终都汇聚到Markdown输出
3. 对于项目结构，要求保持各组件的独立性，不能修改源组件代码
4. 项目应该易于维护和扩展

## 3. 技术选型分析

### 组件功能分析
1. **LibreOffice**
   - 功能：办公套件，可以处理doc/docx到pdf的转换
   - 使用方式：命令行调用
   - 适用格式：doc, docx

2. **Pandoc**
   - 功能：通用文档转换器
   - 使用方式：命令行调用
   - 适用格式：pdf到markdown等多种格式转换

3. **pptx2md**
   - 功能：专门用于将pptx转换为markdown
   - 使用方式：Python脚本或命令行工具
   - 适用格式：pptx

### 转换流程设计
1. doc/docx → (LibreOffice) → pdf → (Pandoc) → md
2. pptx → (pptx2md) → md

## 4. 项目约束和假设

### 技术约束
1. 必须使用指定的三个组件
2. 不能修改源组件代码
3. 需要在项目中fork这些组件到独立路径
4. 项目需要保持简单和纯净

### 假设条件
1. 用户环境中已安装必要的依赖（或项目需提供安装脚本）
2. 输入文件是有效的办公文档格式
3. 磁盘空间足够处理转换过程中的临时文件
